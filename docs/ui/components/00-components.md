# UI Components Library

## Overview

This document defines the reusable UI components used throughout the Chat.buzzi.ai platform. All components follow consistent design patterns, accessibility guidelines (WCAG 2.1 AA), and responsive design principles.

---

## Responsive Patterns

### Breakpoints
| Name | Width | Typical Devices |
|------|-------|-----------------|
| Mobile | <640px | Phones |
| Tablet | 640-1024px | Tablets, small laptops |
| Desktop | 1024-1440px | Laptops, monitors |
| Large | >1440px | Large monitors |

### Responsive Table
- Mobile: Cards or horizontal scroll
- Tablet+: Full table view

### Responsive Navigation
- Mobile: Hamburger menu + bottom nav
- Tablet: Collapsible sidebar
- Desktop: Full sidebar

### Responsive Modals
- Mobile: Full screen
- Tablet+: Centered overlay

---

## Accessibility

### Focus States
All interactive elements have visible focus indicators:
- Outline: 2px solid #0066FF
- Offset: 2px

### Keyboard Navigation
- Tab: Move between interactive elements
- Enter/Space: Activate buttons, checkboxes
- Arrow keys: Navigate within components
- Escape: Close modals, popovers

### ARIA Labels
All icons and non-text elements include:
- `aria-label` for screen readers
- `role` attributes where needed
- `aria-expanded`, `aria-selected` for state

### Color Contrast
All text meets WCAG 2.1 AA requirements:
- Normal text: 4.5:1 minimum
- Large text: 3:1 minimum
- UI components: 3:1 minimum

---

## Animation Guidelines

### Duration
| Type | Duration |
|------|----------|
| Micro (hover, focus) | 150ms |
| Small (tooltips, toasts) | 200ms |
| Medium (modals, dropdowns) | 300ms |
| Large (page transitions) | 400ms |

### Easing
- Enter: `ease-out` (decelerate)
- Exit: `ease-in` (accelerate)
- Move: `ease-in-out`

### Reduced Motion
Respect `prefers-reduced-motion` media query:
- Disable non-essential animations
- Keep essential feedback (focus, loading)
