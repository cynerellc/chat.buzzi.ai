### Payment Fields (Stripe Elements)

| Field | Type | Validation |
|-------|------|------------|
| Card Number | Stripe Card Element | Stripe validation |
| Expiry | Stripe Expiry Element | Valid future date |
| CVC | Stripe CVC Element | 3-4 digits |
| Cardholder Name | Text | Required, 2+ characters |
| Billing Address | Address autocomplete | Country-specific validation |

## Behaviors

### Step Navigation
- **Progress Indicator** - Shows current step, completed steps are clickable
- **Back Button** - Returns to previous step, preserves entered data
- **Form Persistence** - Data saved to session storage between steps
- **Validation** - Each step validated before proceeding

### Email Verification
```
After Step 1 completion:
1. Verification email sent
2. User can continue to Step 2
3. Banner shown: "Please verify your email"
4. Must verify before trial ends
```

### Error Handling

| Error | Display |
|-------|---------|
| Email already exists | Inline: "Email already registered. Sign in instead?" |
| Company name taken | Inline: "Company name not available" |
| Payment declined | Toast: "Payment failed. Please try another card." |
| Network error | Toast with retry option |

### Success Flow
```
1. Payment processed (or trial started)
2. Show success modal
3. Auto-redirect to dashboard (5 seconds)
4. Welcome email sent
```

- Required fields marked with asterisk and `aria-required`
- Password requirements read by screen reader
- Card input accessible via Stripe's accessible elements
