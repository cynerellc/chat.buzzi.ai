# Step 12: Company Admin - Knowledge Base & RAG Pipeline

## Objective
Implement the knowledge base management system including file upload, processing, chunking, embedding, and the RAG (Retrieval Augmented Generation) pipeline for semantic search.

---

## Prerequisites
- Step 11 completed
- Database schema with knowledge_categories, knowledge_files, and knowledge_chunks tables
- Qdrant instance configured and connected
- OpenAI API configured for embeddings

### 12.9 Implement RAG Pipeline

**`src/lib/ai/rag/pipeline.ts`:**

```typescript
// RAG Pipeline Implementation

export interface RagConfig {
  chunkSize: number;        // Default: 500 tokens
  chunkOverlap: number;     // Default: 50 tokens
  topK: number;             // Default: 5 chunks
  similarityThreshold: number; // Default: 0.7
}

export async function processDocument(
  source: KnowledgeSource,
  config?: Partial<RagConfig>
): Promise<void>;

export async function searchKnowledge(
  companyId: string,
  query: string,
  agentId?: string,
  config?: Partial<RagConfig>
): Promise<KnowledgeChunk[]>;
```

**Processing Steps:**
1. Extract text from document (PDF, DOCX, etc.)
2. Clean and normalize text
3. Split into chunks with overlap
4. Generate embeddings for each chunk
5. Store chunks and embeddings in database

  }
): Promise<KnowledgeChunk[]>;

**Qdrant Vector Search:**
```typescript
// Query Qdrant for similar vectors
const collectionName = `company_${companyId.replace(/-/g, '_')}`;
const result = await qdrant.search(collectionName, {
  vector: embedding,
  limit: topK,
  score_threshold: threshold,
  with_payload: true,
});

// Join with PostgreSQL metadata
const chunkIds = result.map(r => r.payload?.file_id);
const metadata = await db.query.knowledgeChunks.findMany({
  where: inArray(knowledgeChunks.qdrantId, chunkIds),
  with: { file: true }
});
```
